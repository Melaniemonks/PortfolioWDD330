Testing and debugging

    Testing and debbuging are done to prevent errors. Errors are caused when something goes wrong in a program. They are usually caused by
    • System error
    • Programmer error
    • User error
    • Exceptions: This is an error that produces a return value that can be used by the program to deal with the error.
    • Stack Traces: An exception will produce a stack trace.
    • Warnings: A warning can occur if there's an error in the code that isn't enough to cause the program to crash.
    • The try statement allows you to define a block of code to be tested for errors while it is being executed.
    • The catch statement allows you to define a block of code to be executed, if an error occurs in the try block.
